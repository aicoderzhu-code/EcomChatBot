║                          Jenkins CI/CD流水线                                   ║
╠════════════════════════════════════════════════════════════════════════════════╣
║                                                                                ║
║  Stage 1: 准备                                                                 ║
║  ├─ 打印构建信息                                                               ║
║  └─ 显示构建编号、分支、提交ID                                                 ║
║                                                                                ║
║  Stage 2: 同步代码                                                             ║
║  ├─ 从Jenkins workspace同步到部署目录                                          ║
║  └─ 排除.git、__pycache__等                                                    ║
║                                                                                ║
║  Stage 3: Docker配置检查                                                       ║
║  └─ 验证docker-compose.yml配置                                                 ║
║                                                                                ║
║  Stage 4: 检查是否需要重建镜像                                                 ║
║  ├─ 检查Dockerfile/requirements.txt是否变化                                    ║
║  └─ 智能判断是否需要重建                                                       ║
║                                                                                ║
║  Stage 5: 构建镜像 (条件执行)                                                  ║
║  ├─ docker-compose build                                                       ║
║  └─ 利用缓存加速构建                                                           ║
║                                                                                ║
║  Stage 6: 部署新服务                                                           ║
║  ├─ docker-compose down (停止旧服务)                                           ║
║  ├─ docker-compose up -d --force-recreate                                      ║
║  └─ 清理未使用镜像                                                             ║
║                                                                                ║
║  Stage 7: 健康检查                                                             ║
║  ├─ 等待服务启动 (30秒)                                                        ║
║  ├─ 检查docker-compose ps状态                                                  ║
║  └─ 检查API健康端点 (http://localhost:8000/docs)                               ║
║                                                                                ║
║  Stage 8: 部署验证                                                             ║
║  ├─ 显示最终服务状态                                                           ║
║  └─ 显示API服务日志                                                            ║
║                                                                                ║
║  ┌──────────────────────────────────────────────────────────────────┐         ║
║  │  Stage 9: 运行自动化测试 ⭐️ 新增                                 │         ║
║  ├──────────────────────────────────────────────────────────────────┤         ║
║  │  执行: ./tests/run_ci_tests.sh                                   │         ║
║  │                                                                  │         ║
║  │  测试脚本功能:                                                   │         ║
║  │  ├─ 检查Python环境                                               │         ║
║  │  ├─ 安装测试依赖 (pytest, httpx, faker等)                        │         ║
║  │  ├─ 等待服务就绪 (健康检查)                                      │         ║
║  │  ├─ 运行测试套件                                                 │         ║
║  │  │   - 300+测试用例                                              │         ║
║  │  │   - 跳过慢速测试 (-m "not slow")                              │         ║
║  │  │   - 并发执行优化                                              │         ║
║  │  ├─ 生成测试报告                                                 │         ║
║  │  │   - JUnit XML (Jenkins集成)                                  │         ║

║  │  └─ 统计测试结果                                                 │         ║
